# AGENTS.md

> Файл поддерживается AI Factory. Обновляйте при значительных изменениях структуры проекта.

## Обзор проекта

Имитационная модель дискретно-событийного моделирования (DES) движения автобусов по маршруту с несколькими остановками. Модель оптимизирует интенсивность выпуска автобусов, максимизируя прибыль при ограничениях времени ожидания пассажиров.

## Технологический стек

- **Язык программирования:** Python 3.11+
- **Библиотеки:** NumPy, SciPy, Matplotlib, JSONSchema
- **Паттерн проектирования:** Дискретно-событийное моделирование (DES)

## Структура проекта

```
AlinaDiplom/
├── src/                      # Исходный код
│   ├── core/                 # Ядро симуляции
│   │   ├── simulation.py     # DES-движок и управление событиями

│   │   └── event.py          # Типы событий (EventType)
│   ├── optimization/         # Алгоритмы оптимизации
│   │   └── optimizer.py      # Поиск оптимальной интенсивности выпуска автобусов
│   ├── visualization/        # Генерация отчетов и графиков
│   │   ├── plotter.py        # Построение визуализаций
│   │   └── console_output.py # Вывод результатов в консоль
│   └── utils/                # Утилиты и хелперы
│       ├── config_loader.py  # Загрузка и валидация конфигурации
│       ├── logger.py         # Настройка логгера
│       └── random_generator.py # Генераторы случайных чисел
├── tests/                    # Модульные тесты
│   ├── test_entities.py      # Тесты сущностей
│   ├── test_config_loader.py # Тесты загрузчика конфигурации
│   └── test_simulation.py    # Тесты симуляции
├── config/                   # Конфигурационные файлы (JSON)
│   ├── example_uniform.json          # Равномерный поток пассажиров
│   ├── example_peak_center.json      # Пиковый трафик в центре
│   ├── example_morning_rush.json     # Утренний час пик
│   ├── example_balanced.json         # Сбалансированная конфигурация
│   ├── example_corrected.json        # Исправленная конфигурация
│   └── example_with_averaging.json   # Конфигурация со средними значениями
├── output/                   # Директория для результатов симуляции
│   ├── logs/                 # Логи выполнения
│   └── plots/                # Графики и визуализации
└── venv/                     # Виртуальное окружение Python
```

## Ключевые точки входа

| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа в программу. Обработка аргументов командной строки, запуск оптимизации |
| `src/core/simulation.py` | Основной класс симуляции (DES-движок), управление очередью событий |
| `src/optimization/optimizer.py` | Алгоритм поиска оптимальной интенсивности выпуска автобусов |
| `config/example_uniform.json` | Пример конфигурации с равномерным потоком пассажиров |

## Правила для AI-агентов

### Декомпозиция команд оболочки

**Правило:** Избегайте комбинирования нескольких команд в одну строку через `&&` или `;`.

- ❌ **Неправильно:** `git checkout main && git pull origin main`
- ✅ **Правильно:** Сначала выполнить `git checkout main`, затем `git pull origin main`

**Обоснование:** Позволяет лучше отслеживать ошибки и упрощает повторное выполнение команд при отладке.
