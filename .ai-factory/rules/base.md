# Базовые правила проекта

> Автоматически обнаруженные соглашения из анализа кодовой базы. При необходимости отредактируйте.

## Соглашения по именованию

- **Файлы:** snake_case (например: `simulation.py`, `config_loader.py`)
- **Переменные:** snake_case (например: `bus_release_intensity`, `passenger_counter`)
- **Функции:** snake_case (например: `_process_passenger_arrival`, `optimize_bus_intensity`)
- **Классы:** PascalCase (например: `Simulation`, `Passenger`, `Bus`, `Stop`)

## Структура модулей

- **src/core/** — ядро симуляции (DES-движок, модели данных, события)
- **src/optimization/** — алгоритмы оптимизации
- **src/visualization/** — генерация отчетов и визуализаций
- **src/utils/** — утилиты и хелперы
- **tests/** — модульные тесты

## Обработка ошибок

- Использование try-except для обработки исключений при загрузке конфигурации
- Логирование ошибок через стандартный логгер `get_logger(__name__)`
- Прерывание выполнения с выводом сообщения об ошибке в консоль

## Логгирование

- Использование настраиваемого логгера из `src.utils.logger`
- Поддержка логирования в консоль и файл
- Уровень логирования управляется параметром `debug_mode` в конфигурации
- Логи сохраняются в поддиректорию `output/logs/`

## Тестирование

- Используется стандартный модуль unittest (предположительно, судя по структуре)
- Тесты расположены в `tests/`
- Именование тестовых файлов: `test_<module>.py` (например: `test_simulation.py`)
- Основные тестируемые компоненты:
  - Сущности (`test_entities.py`)
  - Загрузчик конфигурации (`test_config_loader.py`)
  - Симуляция (`test_simulation.py`)
