# Имитационная модель движения автобусов

## Обзор

Имитационная модель дискретно-событийного моделирования (DES) движения автобусов по маршруту с несколькими остановками. Модель оптимизирует интенсивность выпуска автобусов, максимизируя прибыль при ограничениях времени ожидания пассажиров.

Проект реализует систему массового обслуживания с пуассоновским потоком пассажиров, усечённым нормальным распределением интервалов выпуска автобусов и комплексной метрикой для анализа эффективности работы транспорта.

## Основные возможности

- Дискретно-событийное моделирование (DES)
- Пуассоновский поток пассажиров на остановках
- Усечённое нормальное распределение интервалов выпуска автобусов
- Сбор метрик: время ожидания, длительность поездки, загрузка автобусов
- Автоматическая оптимизация интенсивности выпуска автобусов
- Генерация визуализаций и отчетов
- Поддержка многократных прогонов для статистической достоверности

## Технологический стек

- **Язык программирования:** Python 3.11+
- **Библиотеки:**
  - NumPy — численные вычисления и статистика
  - SciPy — распределения вероятностей
  - Matplotlib — визуализация данных
  - JSONSchema — валидация конфигурации

## Архитектура проекта

### Структура кода

```
src/
├── core/           # Ядро симуляции
│   ├── simulation.py    # Основной класс симуляции (DES-движок)
│   ├── entities.py      # Модели данных: Passenger, Bus, Stop
│   └── event.py         # Типы событий и очередь событий
├── optimization/   # Алгоритмы оптимизации
│   └── optimizer.py     # Поиск оптимальной интенсивности выпуска
├── visualization/  # Генерация отчетов и графиков
│   ├── plotter.py       # Построение визуализаций
│   └── console_output.py # Вывод результатов в консоль
└── utils/          # Утилиты и хелперы
    ├── config_loader.py  # Загрузка и валидация конфигурации
    ├── logger.py         # Настройка логгера
    └── random_generator.py # Генераторы случайных чисел
```

### Основные компоненты

#### Движок симуляции (`src/core/simulation.py`)
- Управление очередью событий (приоритетная очередь)
- Обработка событий: прибытие пассажира, уход пассажира, выпуск автобуса, прибытие автобуса
- Сбор метрик в реальном времени

#### Модели данных (`src/core/entities.py`)
- `Passenger` — пассажир с временем прибытия и ожидания
- `Bus` — автобус с текущей загрузкой и координатами
- `Stop` — остановка с очередью пассажиров и статистикой

#### Оптимизатор (`src/optimization/optimizer.py`)
- Перебор значений интенсивности выпуска
- Многократные прогоны для усреднения результатов
- Поиск точки максимума прибыли

## Поток данных

1. **Загрузка конфигурации** → JSON-файл с параметрами маршрута, экономики и симуляции
2. **Инициализация** → Создание остановок, планирование событий
3. **Цикл симуляции** → Обработка событий по времени
4. **Сбор метрик** → Накопление статистики по всем пассажирам и автобусам
5. **Оптимизация** → Анализ результатов для разных интенсивностей
6. **Визуализация** → Генерация графиков и отчетов

## Метрики симуляции

- Количество обслуженных пассажиров
- Количество потерянных пассажиров (по таймауту ожидания)
- Среднее время ожидания
- Средняя длительность поездки
- Средняя загрузка автобусов
- Доход и расходы → Прибыль
- Процент потерянных пассажиров

## Конфигурация

Конфигурация хранится в JSON-файлах (`config/`):
- `example_uniform.json` — равномерный поток пассажиров
- `example_peak_center.json` — пиковый трафик в центре
- `example_morning_rush.json` — утренний час пик
- `example_balanced.json` — сбалансированная конфигурация
- `example_corrected.json` и `example_with_averaging.json` — дополнительные варианты

## Нетехнические требования

- **Логгирование:** Уровень логирования настраивается через `LOG_LEVEL`
- **Обработка ошибок:** Структурированные сообщения об ошибках для пользовательского ввода
- **Безопасность:** Валидация конфигурации через JSONSchema
