# Командная строка (CLI)

## Базовый синтаксис

```bash
python main.py --config <path> [опции]
```

## Параметры командной строки

### Обязательные параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--config` | Путь к JSON-файлу конфигурации | `--config config/example_uniform.json` |

### Опциональные параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--output` | Директория для сохранения результатов | `output` |
| `--no-plots` | Не создавать графики (отключает генерацию `.png`) | — |
| `--no-save` | Не сохранять результаты в файлы | — |
| `--no-console-logs` | Отключить вывод логов в консоль (только в файл) | — |

## Примеры использования

### Базовый запуск

```bash
python main.py --config config/example_uniform.json
```

### Запуск с кастомной директорией вывода

```bash
python main.py --config config/example_peak_center.json --output results/my-run
```

### Запуск без создания графиков (быстрый прогон)

```bash
python main.py --config config/example_morning_rush.json --no-plots
```

### Запуск в отладочном режиме (с логами только в файл)

```bash
python main.py --config config/example_uniform.json --no-console-logs
```

## Конфигурация через аргументы

Параметры конфигурации JSON могут быть переопределены через код (не CLI). Для изменения:

1. Отредактируйте JSON-файл в `config/`
2. Или создайте новый файл и укажите путь через `--config`

### Примеры конфигов

| Файл | Назначение |
|------|------------|
| `example_uniform.json` | Равномерный поток пассажиров (тестовый) |
| `example_peak_center.json` | Пиковый трафик в центре маршрута |
| `example_morning_rush.json` | Утренний час пик |
| `example_balanced.json` | Сбалансированная конфигурация |
| `example_corrected.json` | Исправленная версия конфигурации |
| `example_with_averaging.json` | Конфигурация со средними значениями |

## Отладка

### Включение отладочного режима

В JSON-конфиге установите:

```json
{
  "simulation": {
    "debug_mode": true,
    "console_logging": false
  }
}
```

### Просмотр логов

Логи сохраняются в `output/logs/simulation_YYYYMMDD_HHMMSS.log`:

```bash
cat output/logs/simulation_20260514_143022.log
# или
type output\logs\simulation_20260514_143022.log  # Windows
```
